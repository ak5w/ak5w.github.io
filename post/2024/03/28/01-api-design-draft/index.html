用户授权

  • 基于 casbin 的 rbac 模型 对所有API 接口进行授权管理。
  • 用户以部门形式被组织着，组织内又可以分配角色，用户也可以直接被以角色进行组织，角色内又才包含了对资源的控制，包括对api 和 前端按钮的访问。
    • jwt 中要包含 user id 和 role id array，从而可以进行授权拦截判断。另外，对外暴露的id 应具有分布式唯一性。此处应使用雪花算法生成。
  • 待定

多租户管理

  • 租户实际上意味着是一个在业务上隔离的应用，应该具备可扩展性，能够低成本脱离多租户，变为单租户平台。
